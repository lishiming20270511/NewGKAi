## 一、产品概述

### 1.1 一句话定位

**为抖音志愿填报主播提供的一站式直播辅助工具**——主播在电脑端操作，画面以竖屏手机比例投屏到直播间，在2步填表中完成考生信息录入→AI分析→付费解锁→生成完整报告。

### 1.2 核心原则（决策引擎宪法）

```
算法负责计算（评分 + 概率）
规则引擎负责约束（过滤 + 分层）  
LLM 负责解释（不参与决策）
```

> 系统目标不是"生成推荐"，而是构建一个**基于历史数据与多因子模型的志愿决策引擎**，输出尽可能贴近真实录取结果的志愿方案。

### 1.3 用户画像（仅2类用户）

| 角色 | 身份 | 核心场景 | 使用方式 |
|------|------|----------|----------|
| **主播** (Streamer) | 抖音高考志愿填报主播 | 直播时帮考生查志愿推荐 | 电脑浏览器打开工具 → 竖屏投屏到抖音直播 |
| **管理员** (Admin) | 平台运营方 | 管理主播账号、充值、监控爬虫 | 管理后台操作 |

> **注：本产品不是 SaaS 多租户系统，是单一平台下的主播工具。一个管理员管理所有主播。**

### 1.4 核心流程

```
主播登录 → STEP1 录入考生基本信息（7个字段）
         → STEP2 填写意向偏好（专业/城市/院校/性格）
         → 点击"开始AI分析"
         → 系统前端纯JS本地生成15所推荐（不依赖后端AI）
         → 展示"付款墙"（关键信息加密）
         → 主播点击"一键解锁" → 后端扣费 → 展示完整报告
         → 可下载PDF / 测下一个考生 / 打开直播答疑
```

### 1.5 商业模式

| 项目 | 说明 |
|------|------|
| 定价 | 299元 = 10次（29.9元/次），管理员可自定义充值次数和金额 |
| 支付方式 | 主播线下转账（微信/支付宝）→ 管理员后台充值次数 |

### 页面⑥：完整报告

**报告总览**：分为 **5 大板块**，A4 竖版编排。

#### 板块一：封面基础信息（封面页）
- 报告编号（`GK` + 年月日时分秒 + 随机码）
- 考生昵称、省份/分数、省内位次、选科、服从调剂、家庭经济
- 数据权威说明：教育部 + 阳光高考 + 2026一分一段
- AI推理引擎：Claude / DeepSeek V4 Pro / GPT-5.5
- 数据库规模：2000+高校、34省、6年2200万+录取记录
- 底部警示：严禁倒卖、仅作参考

#### 板块二：核心定位分析
- 分数段定位 + 全省参考位次 + 意向方向 + 综合竞争力评级
- 院校分层规划：15所 = 5冲刺 + 5稳妥 + 5保底
- 调剂提示

#### 板块三：分层院校明细（15所学校，每校18个信息维度）

| # | 维度 | 说明 |
|:---:|------|------|
| 1 | 学校标签 | 985(金) / 211(紫) / 双一流(青) / 公办 / 民办 |
| 2 | 录取概率 | 百分比 + 颜色编码（红=冲刺/蓝=稳妥/绿=保底） |
| 3 | 城市 | 城市名 + 城市等级（一线/新一线/二线/…） |
| 4 | 近年分数线 | **以2025年为主**（若无则用2024年），禁止编造 |
| 5 | 位次匹配 | ✓位次匹配 或 差距N名 |
| 6 | 分数线趋势 | 逐年分数变化+趋势方向，**展示年份≤5年，按实有数据展示** |
| 7 | **推荐专业** | 按相似度映射，不同学校不同专业名，禁止全校统一 |
| 8 | **学费** | 年学费范围 + 4年总费用 + 家庭承受力判断 |
| 9 | **录取趋势** | 逐年分数变化+趋势方向+描述 |
| 10 | 专业地位 | 国家级/省级一流专业、学科评估等级 |
| 11 | **就业率** | 真实就业率+深造比例，标注数据来源+年份 |
| 12 | **平均薪资** | 应届起薪 + 3年后薪资 |
| 13 | **主要岗位** | 核心岗位(占比%) + 外围岗位(占比%) |
| 14 | 5年趋势 | 行业未来5年岗位需求预测 |
| 15 | **城市分析** | 5子维度：位置/优势/发展/业务/就业影响 |
| 16 | AI点评 | LLM生成，张雪峰风格，口语化，50-100字 |
| 17 | **报考风险分析** | 分差测算（高于录取线×分 / 低于录取线×分）+ 滑档风险等级（低/中/高）+ 调剂建议（建议服从/谨慎考虑/不建议），按冲/稳/保分层标注 |
| 18 | **招生规模趋势** | 近N年计划招生人数 vs 实际录取人数逐年对比（数据源：`plan_count` / `actual_count`）；标注扩招/缩招趋势；规模连续缩减≥2年标注"⚠️ 竞争加剧" |

> **核心原则：每所学校卡片中所有数据必须来自真实爬取结果，不同学校必须呈现差异化内容。禁止全校统一模板填充、禁止编造数据。**

#### PDF 报告规范

**命名格式**：`志愿报告_[网名]_[报告编号].pdf`

**5种水印防伪**（防止主播倒卖）：
| # | 水印类型 | 位置 | 内容 |
|---|----------|------|------|
| 1 | 报告编号 | 封面页眉/正文页眉 | `GK` + 年月日时分秒 + 随机码 |
| 2 | 防伪声明 | 封面底部/正文页眉 | "严禁倒卖 · 本内容由高考志愿规划师提供" |
| 3 | 斜纹水印 | 全页背景（含封面+正文） | "高考志愿规划师 · 严禁倒卖 · [时间戳]"，45°斜角，8%-12%透明度 |
| 4 | 考生信息卡片 | 封面 | 报告编号+生成时间+网名+省份/分数+位次+选科/调剂 |
| 5 | 二维码 | 封面右下角 | 含校验信息的二维码（报告编号+考生信息哈希），扫码可验证真伪 |

**防主播倒卖机制**：
- 每份报告编号写入数据库，关联主播ID + 考生分数 + 省份 + 意向学校哈希
- 新考生报告生成时，后端检测历史报告相似度：
  - 相同省份 + 相同分数段（±5分）+ 相同意向学校 → 标记"疑似重复"
  - 连续3次高度相似 → 后台告警 → 管理员人工核查
- 主播端提示："系统检测到此报告与你 X 分钟前为另一考生生成的报告高度相似，请确认考生信息无误"

### 页面⑧：直播答疑

主播在直播时遇到刁钻问题，快速获取AI回答。

| 特性 | 说明 |
|------|------|
| 快捷问题 | 10个预设问题按钮，一键填入 |
| 自由输入 | 文本框，限100字 |
| AI回答 | 调用LLM，张雪峰风格——犀利、接地气、有数据支撑、200字以内 |
| 历史记录 | 本次会话的问答历史，可清空 |

**预设问题清单**：
1. 这个专业好就业吗？
2. 这所学校值得报吗？
3. 这个专业未来薪资怎么样？

## 四、决策引擎（核心算法）

### 4.1 推荐池构建

推荐结果由两个区域组成：**特别关注区**（意向学校，不计入15所） + **推荐池**（算法生成的15所）。

#### 4.1.1 特别关注区（不计入15所）

| 属性 | 说明 |
|------|------|
| 来源 | 用户填写的意向学校（最多3所） |
| 匹配方式 | 精确名称匹配（`===`），禁止模糊替代 |
| 纳入规则 | **无论概率多少一律展示**；0%概率的学校正常展示，标注"录取概率 0%" |
| 展示位置 | 报告最顶部，标记★，以独立「📌 特别关注」区域展示 |
| 与15所关系 | **不计入15所推荐总数**，推荐池独立填充15所 |

#### 4.1.2 推荐池（15所，四层填充）

| 层级 | 名称 | 来源 | 规则 |
|:---:|------|------|------|
| L1 | **意向城市学校** | 意向城市 → 城市→省份映射（60+城市映射表）→ 该省学校 | 不设概率门槛，标记●强制保留 |
| L2 | **生源地本省学校** | 考生所在省份的学校 | 去除L1已有学校，按rankProb降序取 |
| L3 | **意向城市周边** | 意向城市的地理邻省/邻市学校 | 去除L1/L2已有学校 |
| L4 | **全国兜底** | 邻省 → 沿海发达省份 → 全国 | 补足至15所 |

**优先级总原则**：**意向城市 > 本省 > 意向城市周边 > 全国**

**示例**：河南考生350分，意向城市上海，意向学校北京大学+河南科技大学+洛阳师范学院

```
特别关注区: 北京大学(0%) + 河南科技大学 + 洛阳师范学院 = 3所（不计入15所）
推荐池填充:
  L1: 上海匹配学校 = 4所 → 累计4所
  L2: 河南本省学校 = 补充8所 → 累计12所
  L3: 上海周边(江苏/浙江) = 补充3所 → 累计15所 ✅
```

### 4.2 双推荐率体系

**① 成绩排名推荐率**（`calcRankProb`，录取概率，报告中显著展示，tier分层以此为准）：

```
【Step 1】加权最低位次（多年数据加权平均）

  3年数据齐全：
    加权最低位次 = 2025年最低位次 × 0.5 + 2024年最低位次 × 0.3 + 2023年最低位次 × 0.2

  仅有2年（2025+2024）：
    加权最低位次 = 2025年 × 0.65 + 2024年 × 0.35

  仅有1年（2025）：
    加权最低位次 = 2025年最低位次（100%权重）

  完全无数据：
    rankProb = null，前端显示"暂无录取数据"

【Step 2】位次差值
  位次差值 = 加权最低位次 − 学生省内位次
  （正值 = 学生位次优于历史最低线；负值 = 学生位次差于历史最低线）

【Step 3】概率映射（6档离散值）
  差值 > 5000          → 95%（录取把握大）
  差值 2000 ~ 5000     → 80%
  差值 0 ~ 2000        → 65%
  差值 −2000 ~ 0       → 45%（稳中有险）
  差值 −5000 ~ −2000   → 25%
  差值 < −5000         → 8%（冲刺，风险极高）
```

> **注：** 极低概率直接显示数值（如 8%），不用文字替代"建议不填"。意向学校概率为 0% 时同样展示，标注"成绩未达录取线"。

**② 加权综合推荐率**（`calcWeightedProb`，内部排序用，不作为主要展示值）：

```
总分 = 录取概率(35%) + 专业匹配(20%) + 就业前景(15%)
     + 城市偏好(10%) + 性格匹配(10%) + 经济适应(10%)
```

> 报告同时展示两个推荐率：主展示 `calcRankProb`（六档离散），次展示 `calcWeightedProb`（百分比综合评分），并附简要说明。

### 4.3 Tier 分层

| Tier | 名称 | 成绩排名推荐率范围 | 颜色 |
|:---:|------|:-----------------:|:----:|
| 0 | 🔴 冲刺 | 30% ~ 60% | 红色 |
| 1 | 🔵 稳妥 | 60% ~ 85% | 蓝色 |
| 2 | 🟢 保底 | ≥ 85% | 绿色 |

- <400分考生：阈值降至 5%
- 每tier最多5所
- 非意向学校 rankProb < 30% → 一律不推荐
- **意向学校强制展示，即使 0% 概率也保留**

### 4.4 排序优先级

1. **意向学校**（★ 特别关注区）→ 报告最顶部独立展示
2. **意向城市学校**（● `_intended_city`）→ 推荐池内置顶
3. 同 tier 内按 **rankProb** 降序排列
4. rankProb 相近（差距 ≤ 5%）时，按**性格匹配度**决胜
5. 性格未填写或匹配度相同时，按 **weightedProb** 降序

### 4.5 性格→院校类型倾向（tiebreaker 决胜规则）

| 性格标签 | 优先推荐院校类型 | 逻辑说明 |
|----------|:--------------:|----------|
| 外向活泼、社交沟通、领导管理 | **综合类/文科类** | 擅长人际交往，文科/综合类院校提供更多社交场景 |
| 沉稳内敛、逻辑分析、钻研学术 | **理工科类** | 专注力强、偏好独立钻研 |
| 艺术创作 | **艺术类/设计类** | 匹配艺术类院校 |
| 动手实践 | **工科/应用型** | 偏好实操型学习 |

### 4.6 家庭经济→院校倾向

| 家庭经济 | 专业意向条件 | 优先推荐 | 逻辑说明 |
|----------|-------------|:------:|----------|
| 较为困难 | 专业方向含师范/教育类 | **师范类院校** | 师范类有生活费补贴、免学费/低学费政策 |
| 较为困难 | 无特定师范意向 | **学费≤5000元/年公办院校** | 标注"经济友好" |
| 一般/良好/比较优越 | — | 无特殊限制 | 按正常权重评分 |

### 6.2 产品经理建议

**推荐方案 A（Bento Grid 暗色）作为生产版本，理由如下：**

| # | 决策依据 | 分析 |
|---|---------|------|
| 1 | **直播压缩表现** | 暗色背景经 H.264 4:2:0 采样后结构保留完整；紫色渐变在低码率下会出现明显的"色带/banding"伪影 |
| 2 | **信息可读性** | 暗底亮字天然对比度高；方案B的白色内容面板在紫色背景上"浮起"，观众容易视觉跳脱 |
| 3 | **专业信任感** | 对标产品（radar.lumenaistudio.co）为深色系；Bloomberg Terminal/Stripe Dashboard 均为暗色——家长看到暗色界面会本能认为"这是专业工具" |
| 4 | **主播体验** | 长时间直播（3-4小时），暗色界面不刺眼；方案B的紫色渐变长时间注视可能导致色彩疲劳 |

> ✅ **产品决策（2026-06-18 确认）**：方案 A（Bento Grid 暗色主题）为正式生产 UI，**方案 B（紫色渐变）暂缓**，待 Bug 清零后视情况启动原型验证（需在真实直播推流环境下测试后再决定是否切换）。

### 场景 3：低分考生 + 意向城市跨省

> **输入**：河南考生350分，意向城市上海，意向学校北京大学+河南科技大学+洛阳师范学院
>
> **预期结果**：
> - 特别关注区展示3所意向学校（不计入15所）；北京大学标注"录取概率 0% · 成绩未达录取线"
> - 推荐池15所按四层填充：L1上海学校（优先）→ L2河南本省学校（补足）→ L3上海周边（江苏/浙江）→ L4全国兜底
> - 低分段（<400分）阈值降至5%，专科/民办院校纳入推荐范围

### 场景 4：算法边界与数据缺失

> **预期结果**：
> - rankProb < 30% 的非意向学校不进入推荐池，前端不展示
> - 某校录取数据仅有1年时：算法使用该年100%权重，并在卡片中注明"仅有1年历史数据，参考价值有限"
> - 完全无录取数据时：rankProb = null，前端显示"暂无录取数据"
> - 招生规模连续≥2年缩减时：维度18标注"⚠️ 竞争加剧"

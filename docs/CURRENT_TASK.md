# 当前任务

## TASK001: 推荐算法概率计算优化

### 修改文件
- `api/services/recommendation.py`

### 修改函数
1. `_tier_score_prior()` — 降低 985/211 先验概率值（降低约 40-50%）
2. `calc_rank_prob()` — 将混合比例从 `0.65*data + 0.35*prior` 改为 `0.90*data + 0.10*prior`
3. `sort_and_slice()` — 限制冲刺档回填 `globe_expanded` 学校数量（最多 2 所），排除在稳妥/保底档回填
4. `generate_recommendation()` — 位次缺失时使用层次先验概率替代硬编码 50%
5. `sort_and_slice()` — 回填时按分数段应用质量过滤（高分段冲刺仅保留 985/211/双一流，稳妥/保底排除专科）
6. `generate_recommendation()` — 缓存键中 `personality` 和 `economic` 使用完整值而非排序值

### 修复问题
| 问题 | 严重性 | 影响范围 | 修复方式 |
|------|--------|----------|----------|
| 先验概率过高导致冲刺档被淹没 | 高 | 所有推荐 | 降低先验概率值 + 混合比例改为 90% 数据驱动 |
| L4 全国扩展学校错误分配到保底档 | 中 | 冲刺档 | 限制冲刺档回填最多 2 所，排除在稳妥/保底档 |
| 位次估算失败时概率计算退化 | 高 | 无数据省份 | 使用层次先验概率替代硬编码 50% |
| 回填时质量过滤缺失 | 中 | 高分段考生 | 回填时按分数段应用质量过滤 |
| 缓存键不完整 | 低 | 缓存命中 | 使用完整值而非排序值 |

### 验证方法
1. 启动服务后调用 `POST /api/recommendation/generate` 端点
2. 检查返回结果中 `tier_summary` 的 `boost`（冲刺）数量是否 ≥ 3
3. 检查高分段考生（≥500 分）的冲刺档是否仅包含 985/211/双一流院校
4. 检查无位次数据省份（如西藏）的推荐结果是否合理
5. 检查不同经济水平/性格的考生是否得到不同的缓存结果

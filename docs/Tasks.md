### TASK-10：broadcast_scripts 表 + 话术 CRUD API（后端）

| 属性 | 内容 |
|------|------|
| **优先级** | P1 |
| **状态** | ✅ 已完成（v5.23，`api/routers/broadcast_scripts.py` 含8接口，main.py 已注册，DB `broadcast_scripts` 表已建，15条默认话术已导入） |

| **文件** | `api/routers/broadcast_scripts.py`；`main.py` |

**工作内容**：
1. 确认/新建 `broadcast_scripts` 表：
   ```sql
   CREATE TABLE broadcast_scripts (
       id INT AUTO_INCREMENT PRIMARY KEY,
       category VARCHAR(64) NOT NULL COMMENT '分类名',
       title VARCHAR(128) NOT NULL COMMENT '话术标题',
       content TEXT NOT NULL COMMENT '话术正文',
       sort_order INT NOT NULL DEFAULT 0 COMMENT '同分类内排序',
       status ENUM('active','disabled') NOT NULL DEFAULT 'active',
       created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
       updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
       INDEX idx_category (category),
       INDEX idx_sort (category, sort_order)
   ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
   ```
2. 管理员接口（`/admin/scripts/`，Admin JWT）：
   - `GET /admin/scripts` — 分类分组列表
   - `POST /admin/scripts` — 新增话术（category, title, content）
   - `PUT /admin/scripts/{id}` — 编辑话术
   - `DELETE /admin/scripts/{id}` — 删除话术
   - `PUT /admin/scripts/{id}/sort` — 更新排序
   - `GET /admin/scripts/categories` — 获取所有分类名
3. 主播读取接口（`/api/scripts`，主播 JWT）：`GET /api/scripts` — 分类分组列表
4. 预置 5 个默认分类：开播话术 / 产品介绍 / 报告讲解 / 收单话术 / 常见异议处理

**验收**：`GET /api/scripts` 返回正确分类结构；CRUD 幂等正确；Admin JWT 限制生效

### TASK-15：高分保底分差校验（后端算法）

| 属性 | 内容 |
|------|------|
| **优先级** | P0 |
| **状态** | ✅ 已完成（v5.6） |
| **预计工时** | 2–3 h |
| **文件** | `api/services/recommendation.py` |

**需求来源**：Bug报告"683分高分考生保底院校分数线严重脱节，梯度逻辑失效"

**现象**：683分考生保底推荐广东海洋大学(544分)、韶关学院(526分)、岭南师范学院(535分)、北京城市学院(480分)、上海杉达学院(480分)，分差最大203分，跨越两个完整分数段。

**根因**：`apply_quality_threshold_filter` 对高分段（≥一本线）的保底档仅要求"有一流专业的公办二本"，没有区分普通高分(620-650)与顶尖高分(670+)，导致所有高概率低分院校均可通过过滤进入保底推荐。

**工作内容**：
1. 定义常量（放在文件顶部配置区，与其他阈值常量并列）：
   ```python
   ULTRA_HIGH_SCORE_THRESHOLD = 670   # 超高分段起始线
   ULTRA_HIGH_SCORE_FLOOR = 680       # 硬拦截起始线
   ULTRA_HIGH_BOTTOM_GAP = 50         # 超高分段保底最大分差
   ULTRA_HIGH_HARD_MIN_SCORE = 600    # 硬拦截：保底院校最低录取分下限
   ```
2. 在 `apply_quality_threshold_filter()` 函数中，高分段保底档（tier=2）判断后新增超高分段分差校验：
   ```python
   # 超高分段保底分差校验（在高分段 tier=2 过滤之后追加）
   if student_score >= ULTRA_HIGH_SCORE_THRESHOLD:
       weighted_min = school.weighted_min_score  # 加权最低分（已在SchoolRecord中）
       if weighted_min is None:
           pass  # 无数据不过滤
       elif student_score >= ULTRA_HIGH_SCORE_FLOOR and weighted_min < ULTRA_HIGH_HARD_MIN_SCORE:
           return False  # 硬拦截：680+分 → 剔除历史最低分<600的院校
       elif student_score - weighted_min > ULTRA_HIGH_BOTTOM_GAP:
           return False  # 软拦截：分差>50 → 剔除
   ```
3. 该分差校验仅对 tier=2（保底）档的非意向院校生效；意向院校（特别关注区）豁免
4. 超高分段保底被全部剔除时（保底数 < 2）：将分差阈值放宽至 70 重跑一次，避免保底为空
5. `SchoolRecord` 新增字段 `weighted_min_score: Optional[float] = None`，在 `build_candidate_pool` 构建候选时从 `admission_history` 加权计算填入（复用 `calcRankProb` 已有的加权最低位次计算逻辑）

**验收**：
- 683分考生：保底院校历史最低录取分 ≥ 633（683-50）；广东海洋大学(544)、韶关学院(526) 不出现
- 600分考生：原有保底逻辑不受影响（公办强势二本依然可纳入保底）
- 所有意向院校无论分差大小仍出现在特别关注区

> ⚠️ **审批要求**：修改 recommendation.py 核心逻辑前须在对话中获得用户明确同意

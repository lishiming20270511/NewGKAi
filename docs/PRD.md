## 一、产品概述

### 1.1 一句话定位

**为抖音志愿填报主播提供的一站式直播辅助工具**——主播在电脑端操作，画面以竖屏手机比例投屏到直播间，在2步填表中完成考生信息录入→AI分析→付费解锁→生成完整报告。

**访问地址**：`http://121.41.69.234`（新服务器，纯HTTP）

> **为何用IP而非域名**：域名购自国外注册商，阿里云对未备案境外域名实施HTTP拦截（ICP备案限制），故采用IP直连方式。主播为内部用户非公开注册，HTTP明文传输在安全组端口限制下风险可控。

### 1.2 用户画像（仅2类用户）

| 角色 | 身份 | 核心场景 | 使用方式 |
|------|------|----------|----------|
| **主播** (Streamer) | 抖音高考志愿填报主播 | 直播时帮考生查志愿推荐 | 电脑浏览器打开工具 → 竖屏投屏到抖音直播 |
| **管理员** (Admin) | 平台运营方 | 管理主播账号、充值、监控爬虫 | 管理后台操作 |

> **注：本产品不是 SaaS 多租户系统，是单一平台下的主播工具。一个管理员管理所有主播。**

### 1.3 爬虫系统说明

- **爬虫运行位置**：海外独立服务器（`199.193.126.80`）
- **数据同步方式**：爬虫获取的数据通过 SSH 隧道同步到生产服务器 MySQL
- **爬虫不暴露给主播**：主播感知不到爬虫的存在，只管用数据
- **爬虫任务体系**：系统自动检测数据缺口，通过以下任务表驱动爬虫工作：

| 任务表 | 用途 | 触发时机 |
|--------|------|---------|
| `school_admission_crawl_tasks` | 录取数据爬取（历年分数线/位次） | 考生查询某校时发现数据缺失 |
| `school_major_crawl_tasks` | 专业数据爬取（该校开设专业列表） | 推荐引擎发现该校缺少专业→相近专业映射数据 |
| `school_tuition_crawl_tasks` | 学费数据爬取（各专业学费标准） | 冷门院校首次被推荐时触发 |
| `school_employment_crawl_tasks` | 就业数据爬取（就业率/薪资/岗位） | 推荐引擎发现该校就业数据为空或过期（>1年） |
| `school_salary_crawl_tasks` | 薪资数据爬取（应届+3年后薪资） | 同上 |
| `school_city_crawl_tasks` | 城市分析数据（城市5维信息） | 首次出现该城市时触发，后续缓存复用 |

**数据采集优先级**：录取分数 > 专业列表 > 学费 > 就业率 > 薪资 > 城市分析

### 1.4 核心流程（对标 radar.lumenaistudio.co）

```
主播登录 → STEP1 录入考生基本信息（7个字段）
         → STEP2 填写意向偏好（专业/城市/院校/性格）
         → 点击"开始AI分析"
         → 系统前端纯JS本地生成15所推荐（不依赖后端AI）
         → 展示"付款墙"（关键信息加密）
         → 主播点击"一键解锁" → 后端扣费 → 展示完整报告
         → 可下载PDF / 打开直播答疑
```

### 页面④：分析中

过渡动画页面——显示加载状态（spinner + "AI分析中…"），约1-2秒后自动跳转到付款墙。

**背后发生的事情**：
- 前端 `generateSchools()` 在浏览器本地执行推荐算法（纯JS，不依赖后端AI）
- 特别关注区构建（意向学校独立提取）
- 四层填充（意向城市→本省→意向城市周边→全国）
- 位次概率计算
- Tier分层（冲/稳/保）
- 15所学校排序

> **设计原则**：推荐由算法+规则完成，LLM只负责解释（生成AI点评文字），不参与决策。

#### 板块二：核心定位分析（报告第2页）

##### 2.1 分数定位

| 维度 | 内容 |
|------|------|
| 分数段定位 | 普通本科稳妥段 |
| 全省参考位次 | 约 360,000 名 |
| 意向方向 | 医学 / 生命科学类 |
| 综合竞争力评级 | 中段竞争力，建议以稳妥+保底为主 |

##### 2.2 院校分层规划

总计 **15 所**精选院校，分为三个梯度：

| Tier | 名称 | 数量 | 录取概率范围 | 填报策略 |
|:---:|------|:---:|:-----------:|---------|
| 🔴 | **冲刺院校** | 5 所 | 22% ~ 38% | 搏一搏，不依赖 |
| 🔵 | **稳妥院校** | 5 所 | 52% ~ 72% | 主力填报，重点研究 |
| 🟢 | **保底院校** | 5 所 | 83% ~ 94% | 确保有学上 |

##### 2.3 调剂提示

服从调剂可提升录取概率约 8%-12%，显著降低滑档风险。建议除冲刺院校外，稳妥和保底院校均勾选"服从调剂"。

| **数据缺口处理** | 若缺少某校专业数据，标记为"数据获取中…"并创建爬虫任务（`school_major_crawl_tasks` 表） |
| **相近专业映射** | 系统内置专业相似度映射表（如 通信工程↔信息工程/电子信息工程/电子科学与技术），用于自动匹配最佳替代专业 |

#### 维度8：学费（详细规则）

| 规则 | 说明 |
|------|------|
| **数据来源** | 各高校官网/招生章程中的真实学费标准 |
| **爬取策略** | 混合策略：985/211/双一流院校提前全量爬取入库；普通院校首次被推荐时后台异步爬取，卡片先展示"学费查询中…"占位，爬取完成后动态刷新 |
| **禁止造假** | 不同学校不得显示完全相同的学费。每学校×专业组合的学费独立存储于 `school_tuition` 表 |
| **家庭经济关联** | 学费≤5000元/年标记"经济友好 💚"；学费与其他学校的差异必须在报告中如实体现 |
| **数据格式** | `年学费范围 + 4年总费用 + 家庭承受力一句话判断` |

#### 维度11：就业率（详细规则）

| 规则 | 说明 |
|------|------|
| **数据来源优先级** | ① 教育部《高校毕业生就业质量年度报告》（2025年版，若无则取2024年版）；② 该校官网就业质量报告PDF；③ 第三方平台（猎聘/智联/Boss直聘公开报告）+ 多源交叉验证 |
| **按学校差异化** | 每所学校的就业率数据必须不同，存储于 `school_employment` 表 |
| **数据标注** | 每条就业数据标注来源和年份（如"数据来源：郑州大学2025届毕业生就业质量报告"） |
| **禁止造假** | 禁止全校使用统一模板填充；若某校数据暂缺，先展示"就业数据获取中…"并创建爬虫任务 |

#### 维度12：平均薪资（详细规则）

| 规则 | 说明 |
|------|------|
| **数据来源优先级** | ① 教育部报告（官方薪资统计通常不披露）；② 该校就业质量报告；③ 第三方招聘平台交叉验证（猎聘/智联/看准网/职友集） |
| **按学校差异化** | 每所学校相同专业的薪资数据必须独立，不同学校薪资水平应体现城市/学校层次差异 |

| **禁止造假** | 不同学校的薪资数据不得完全相同 |

|---|--------|---------|------|

## 三、决策引擎（核心算法）

### 3.1 算法原则

```
算法负责计算（评分 + 概率）
规则引擎负责约束（过滤 + 分层）
LLM 负责解释（不参与决策）
```

前端 `generateSchools()` 纯JS执行，不依赖后端AI。

### 3.2 推荐池构建

推荐结果由两个区域组成：**特别关注区**（意向学校，不计入15所） + **推荐池**（算法生成的15所）。

#### 3.2.1 特别关注区（不计入15所）

| 属性 | 说明 |
|------|------|
| 来源 | 用户填写的意向学校（最多3所） |
| 匹配方式 | 精确名称匹配（`===`），禁止模糊替代 |
| 纳入规则 | **无论概率多少一律展示**；0%概率的学校正常展示，标注"录取概率 0%" |
| 展示位置 | 报告最顶部，标记★，以独立「📌 特别关注」区域展示 |
| 与15所关系 | **不计入15所推荐总数**，推荐池独立填充15所（5冲刺+5稳妥+5保底） |

**设计理由**：意向学校是考生的"心愿清单"，不应挤占算法推荐名额。0%概率的学校（如350分报北大）如实展示，让主播和考生直面现实，同时不影响正常推荐的tier结构。

#### 3.2.2 推荐池（15所，四层填充）

15所推荐学校按以下优先级逐层填充，每层去重：

| 层级 | 名称 | 来源 | 规则 |
|:---:|------|------|------|
| L1 | **意向城市学校** | 意向城市 → 城市→省份映射（60+城市映射表）→ 该省学校 | 不设概率门槛，标记●强制保留；优先从该城市所在省份的学校中按rankProb降序选取 |
| L2 | **生源地本省学校** | 考生所在省份的学校 | 去除L1已有学校，按rankProb降序取 |
| L3 | **意向城市周边** | 意向城市的地理邻省/邻市学校 | 去除L1/L2已有学校，按rankProb降序取 |
| L4 | **全国兜底** | 邻省 → 沿海发达省份 → 全国 | 去除L1/L2/L3已有学校，按rankProb降序填充至15所 |

**填充逻辑说明**：
- 总推荐数**固定为15所**（尽量维持5冲刺+5稳妥+5保底，当某tier不足时从L4按分数接近度补足）
- L1 意向城市学校优先占位
- 若 L1 < 15，L2 从本省补充
- 若 L1+L2 < 15，L3 从意向城市周边补充
- 若 L1+L2+L3 < 15，L4 从全国兜底

**优先级总原则**：**意向城市 > 本省 > 意向城市周边 > 全国**

**示例**：河南考生350分，意向城市上海，意向学校3所
```
特别关注区: 北京大学(0%) + 河南科技大学 + 洛阳师范学院 = 3所（不计入15所）
推荐池填充:
  L1: 上海匹配学校 = 4所 → 累计4所
  L2: 河南本省学校 = 补充8所 → 累计12所
  L3: 上海周边(江苏/浙江) = 补充3所 → 累计15所 ✅
```

### 3.3 概率计算（双推荐率）

**① 成绩排名推荐率**（`calcRankProb`，tier分层以此为准）：
```
位次比值 = 学生省内位次 / 该校历年最低位次
概率 = (1 - 位次比值) × 100 + 随机扰动(±3%)
钳制到 [1, 99]
```

**② 加权综合推荐率**（`calcWeightedProb`，六维度加权）：
```
总分 = 录取概率(35%) + 专业匹配(20%) + 就业前景(15%)
     + 城市偏好(10%) + 性格匹配(10%) + 经济适应(10%)
```

### 3.4 Tier分层

| Tier | 名称 | 成绩排名推荐率范围 | 颜色 |
|:---:|------|:-----------------:|:----:|
| 0 | 🔴 冲刺 | 30% ~ 60% | 红色 |
| 1 | 🔵 稳妥 | 60% ~ 85% | 蓝色 |
| 2 | 🟢 保底 | ≥ 85% | 绿色 |

- <400分考生：阈值降至5%
- 每tier最多5所（基础版）

### 3.5 排序优先级

1. **意向学校**（★ 特别关注区）→ 报告最顶部独立展示，不参与排序
2. **意向城市学校**（● `_intended_city`）→ 推荐池内置顶
3. 同 tier 内按 **rankProb** 降序排列
4. rankProb 相近（差距 ≤ 5%）时，按**性格匹配度**决胜（见3.6.1）
5. 性格未填写或匹配度相同时，按 **weightedProb** 降序

---

### 3.6 性格与经济偏好匹配

在**加权综合推荐率（weightedProb）**中，性格匹配(10%)与经济适应(10%)已占权重（见3.3节）。以下为具体匹配规则，指导六维度评分计算，并作为**同分决胜（tiebreaker）**依据。

#### 3.6.1 性格→院校类型倾向

当两所学校 rankProb 差距 ≤ 5% 时，性格匹配度决定先后顺序：

| 性格标签 | 优先推荐院校类型 | 逻辑说明 |
|----------|:--------------:|----------|
| 外向活泼、社交沟通、领导管理 | **综合类/文科类** | 擅长人际交往，文科/综合类院校提供更多社交场景与人文学科资源 |
| 沉稳内敛、逻辑分析、钻研学术 | **理工科类** | 专注力强、偏好独立钻研，理工科环境更匹配 |
| 艺术创作 | **艺术类/设计类** | 匹配艺术类院校或综合大学的艺术/设计专业 |
| 动手实践 | **工科/应用型** | 偏好实操型学习，工科院校的实验室/实训资源更丰富 |

> **院校类型判定**：优先读取数据库 `school_type` 字段；若无此字段，按学校名称关键词判定（"师范"→师范类，"理工/工业/科技/交通"→理工科类，"艺术/美术/音乐/戏剧"→艺术类，"财经/政法/语言/外国语"→文科类，其余→综合类）。

#### 3.6.2 家庭经济→院校类型倾向

| 家庭经济 | 专业意向条件 | 优先推荐 | 逻辑说明 |
|----------|-------------|:------:|----------|
| 较为困难 | 专业方向含师范/教育类标签 | **师范类院校** | 师范类院校普遍有生活费补贴、免学费/低学费政策，可显著减轻家庭负担 |
| 较为困难 | 无特定师范/教育类意向 | **学费较低的公办院校** | 优先推荐学费 ≤ 5000元/年的公办院校，标注"经济友好" |
| 一般/良好/比较优越 | — | 无特殊限制 | 按正常权重评分 |

**师范类院校识别**：学校名称含"师范"二字，或数据库 `school_type` 字段标记为"师范"。当前数据库至少有：北京师范大学、华东师范大学、东北师范大学、华中师范大学、陕西师范大学、西南大学（原西南师范大学）等6所部属师范 + 各省属师范院校。

#### 3.6.3 专业方向→院校匹配

若考生填写了专业方向（26个标签多选），在计算加权综合推荐率的专业匹配(20%)得分时：

| 条件 | 得分 |
|------|:---:|
| 该校该专业为国家一流专业建设点 | 满分（20%） |
| 该校该专业为省级一流专业建设点 | 15% |
| 该校有该专业但非一流 | 10% |

## 六、数据流程

### 6.1 核心业务流

```
① 登录
  主播输入手机号+密码 → POST /auth/login → JWT token + 剩余次数
                                    │
② 录入考生信息                   ↓
  抖音昵称+省份+分数+位次(可选)+选科+服从调剂+家庭经济
                                    │
③ 意向偏好                      ↓
  专业方向+城市+院校+性格标签
                                    │
④ 开始AI分析                    ↓
  后端 POST /api/recommendation/generate：
  - 位次估算（查询一分一段表）
  - 特别关注区构建（意向学校独立提取，不计入15所）
  - 四层填充（意向城市 → 本省 → 意向城市周边 → 全国）
  - 概率计算（成绩排名推荐率 + 加权综合推荐率）
  - Tier分层（冲30-60% / 稳60-85% / 保≥85%）
  - 性格匹配决胜（rankProb差距≤5%时触发）
  - 排序（意向城市●置顶 → rankProb降序 → 性格tiebreaker）
  - 数据缺口检测（见6.3）→ 自动创建爬虫任务
                                    │
⑤ 付款墙                        ↓
  4项AI洞察遮蔽 + 15所学校加密
  显示剩余次数 + 一键解锁按钮
                                    │
⑥ 扣费 + 完整报告               ↓
  POST /auth/deduct → balance-1
  展示15所学校完整卡片（16个信息维度）
  - 真实学费数据（热门已爬取即时展示，冷门异步加载）
  - 真实就业率/薪资（按来源标注年份）
  - 真实推荐专业（按相似度映射）
  - 城市5维分析
  LLM生成点评（口语化风格）
```

### 6.2 扣费流程（原子操作）

```
前端 doDirectUnlock():
  ① 乐观扣减：localBalance -= 1，更新剩余次数UI
  ② POST /auth/deduct (Authorization: Bearer ***
  ③ 后端：
     BEGIN TRANSACTION
       SELECT balance FROM streamer_accounts WHERE id=? FOR UPDATE
       IF balance < 1 → ROLLBACK，返回400
       UPDATE balance = balance - 1, used_total = used_total + 1
     COMMIT
     返回 { balance, used_total }
  ④ 成功(200) → 确认前端状态，展示完整报告
  ⑤ 失败(400/403/500) → 回滚 localBalance += 1，toast提示
```

### 6.3 数据缺口检测与爬虫任务

推荐引擎生成报告时，自动检测每所学校的16个维度数据完整性：

```
推荐引擎 generate()
   │
   ├─ 维度4: 近年线数据 → 查询 admission_history (2025年)
   │    └─ 缺失 → 创建 school_admission_crawl_tasks
   │
   ├─ 维度6-9: 趋势/录取数据 → 查询 admission_history (近5年)
   │    └─ 年份不足 → 创建 school_admission_crawl_tasks (补足缺失年份)
   │
   ├─ 维度7: 推荐专业 → 查询 school_majors + major_similarity 映射
   │    └─ 该校专业数据缺失 → 创建 school_major_crawl_tasks
   │
   ├─ 维度8: 学费 → 查询 school_tuition
   │    ├─ 热门院校(985/211/双一流): 应已爬取，若缺失 → 告警
   │    └─ 冷门院校: 首次被推荐 → 创建 school_tuition_crawl_tasks
   │
   ├─ 维度11: 就业率 → 查询 school_employment
   │    └─ 数据为空或年份<2024 → 创建 school_employment_crawl_tasks
   │
   ├─ 维度12: 薪资 → 查询 school_salary
   │    └─ 数据为空或年份<2024 → 创建 school_salary_crawl_tasks
   │
   └─ 维度15: 城市分析 → 查询 city_analysis (缓存)
        └─ 首次出现该城市 → 创建 school_city_crawl_tasks
```

**前端处理**：
- 已爬取数据维度：正常展示
- 待爬取数据维度：展示"数据获取中…"骨架屏占位
- 爬虫完成后，通过 WebSocket / 轮询刷新卡片对应维度

**爬虫执行**（海外服务器 199.193.126.80）：
```
cron(每5分钟) → 查询各 crawl_tasks 表中 pending 任务
  → 按优先级执行爬取（录取分数 > 专业 > 学费 > 就业 > 薪资 > 城市）
  → POST /internal/crawler/ingest → crawler_staging → 校验 → MERGE 入库
  → 更新任务状态为 done / failed
```

| 类别 | 要求 |
|------|------|

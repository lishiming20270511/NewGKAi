## 一、设计总则

### 1.1 核心设计原则

| 原则 | 说明 |
|------|------|
| **竖屏优先** | 375px 宽，模拟手机比例，适配抖音直播投屏 |
| **固定高度** | 手机容器强制 `height: 800px`，**永不随内容增长**。超出内容通过 `.page-container` 内部滚动展示，顶部导航栏和底部操作栏始终固定可见 |
| **大字体/高对比度** | 最小字号 14px，关键数据 ≥ 18px 加粗，WCAG AA 对比度 |
| **CTA 统一** | 所有主流程操作按钮（登录/下一步/解锁）统一使用红色 `btn-danger`（`--color-danger`），与登录页风格一致 |
| **手机模拟框** | 所有主播端页面包裹在 `.phone-frame` 内（顶部状态栏 + 底部 Home Bar） |
| **即时反馈** | 加载<200ms 直接渲染，200ms-1s 显示骨架屏，>1s 显示 spinner |
| **颜色即信息** | 红=冲刺/CTA/危险，蓝=稳妥/信息，绿=保底/成功，灰=未解锁/禁用 |

## 二、全局组件

### 2.1 手机模拟框 `.phone-frame`

**布局约束（直播适配关键）**：

```
┌──────────────────────────┐
│ 📶 9:41          🔋 ⚡  │ ← 状态栏 (28px, flex-shrink:0)
├──────────────────────────┤
│ [导航栏按钮…]   [🔴直播] │ ← 导航栏 (flex-shrink:0, sticky)
├──────────────────────────┤
│                          │
│    .page-container       │ ← flex:1 + overflow-y:auto
│    (内容区，可内部滚动)    │    高度 = 800 - 28 - nav - 24
│                          │
│                          │
├──────────────────────────┤
│          ────            │ ← Home Bar (24px, flex-shrink:0)
└──────────────────────────┘
     ↑ 375 × 800px 强制固定，永不增长
```

**CSS 关键约束**：
| 属性 | 值 | 说明 |
|------|-----|------|
| `width` | `375px` | 手机比例宽度 |
| `height` | `800px` | **强制固定**，不可用 min-height |
| `overflow` | `hidden` | 截断超出内容 |
| `display` | `flex; flex-direction: column` | 垂直弹性布局 |
| `.status-bar` | `flex-shrink: 0` | 固定不压缩 |
| `.nav-bar` | `flex-shrink: 0` | 固定不压缩 |
| `.page-container` | `flex: 1; overflow-y: auto` | 填充剩余空间，内部滚动 |
| `.home-bar` | `flex-shrink: 0` | 固定不压缩 |

> **设计意图**：直播投屏时，手机框高度必须在屏幕内完整可见。内容再多也不能把容器撑长，只能在内部滚动。顶部导航和底部按钮始终保持可见。

### 2.2 快速导航栏 `.nav-bar`

全局顶部导航，9个按钮 + 直播模式切换：

| 按钮 | 图标 | 目标页面 | 状态 |
|------|:---:|------|:---:|
| 登录 | 🔑 | page-login | 登录后隐藏 |
| 考生信息 | ① | page-student | — |
| 意向偏好 | ② | page-pref | STEP1 未完成时置灰 |
| 分析中 | ⏳ | page-analyzing | 自动跳转，不可手动 |
| 付款墙 | 🔒 | page-paywall | 自动跳转，不可手动 |
| 完整报告 | 📊 | page-report | 解锁后才可见 |
| 报告样板 | 📋 | page-sample | 无需登录 |
| 直播答疑 | 💬 | page-qa | 需登录 |
| [直播模式] | 🔴 | toggle | 全局切换 |

> **📌 页面编号说明**：导航栏中 ①-⑥ 为主流程页面（登录→考生信息→意向偏好→分析中→付款墙→完整报告）。页面 ⑦（报告样板）和 ⑧（直播答疑）为辅助功能页。页面 ⑨（直播模式）为全局切换功能，以「🔴 直播模式」按钮置于导航栏右侧，非独立页面。导航栏上「⑧报告样板」「⑨直播答疑」为历史编号遗留，对应文档中「页面⑦（报告样板）」和「页面⑧（直播答疑）」，前端实现时导航按钮文案使用中文名称而非编号。

### 2.3 Toast 通知 `.toast`

| 类型 | 颜色 | 图标 | 持续 |
|------|------|:---:|:---:|
| 成功 | `#22C55E` | ✓ | 3s |
| 错误 | `#EF4444` | ✕ | 5s 或点击关闭 |
| 警告 | `#F59E0B` | ⚠ | 5s |
| 信息 | `#3B82F6` | ℹ | 3s |

位置：顶部居中，距手机框顶部 16px，动画 ease-out 300ms。

### 2.4 模态框 `.modal`

居中对齐，半透明黑色遮罩 `rgba(0,0,0,0.5)`，白色卡片圆角 12px，padding 24px。

### 页面：分析中 (`page-analyzing`)

**设计原则**：浅色科技风，与整体设计协调统一。通过数据网格底纹+流动扫描线+渐变色标题+三引擎徽章来营造技术感，而非使用独立的深色主题。

**页面底色**：浅蓝渐变 `linear-gradient(180deg, #F5F7FA 0%, #EEF2FF 50%, #F5F7FA 100%)`

| 层级 | 组件 | 类型 | 说明 |
|:---:|------|------|------|
| 背景 | 数据网格 | CSS伪元素 | `radial-gradient` 24px间距蓝点网格，`rgba(59,130,246,0.06)` |
| 背景 | 扫描线 | CSS伪元素 | 蓝紫渐变横线从上到下 3s 循环移动 |
| 中心 | 脑图标 | Icon | 🧠 emoji，56px圆角方块，`#EEF2FF→#E0E7FF` 渐变底 + 虚线扫描边框动画 |
| 中心 | 标题 | Text | 「⚡ AI 深度分析中」，17px/800，渐变色 `#3B82F6→#6366F1` |
| 中心 | 三引擎徽章 | Badge×3 | Claude(蓝点) / DeepSeek V4(紫点) / GPT-5.5(青点)，白色圆角标签 |
| 进度区 | 步骤1-2 | StepCard×2 | 白色卡片，渐变色滑入背景动画，active=蓝色渐变/蓝字，done=绿色底/绿字 |
| 信息卡 | 数据基础 | InfoCard | 📂 白色卡片，2000+高校·34省·2200万+记录 |
| 信息卡 | 分析引擎 | InfoCard | 🤖 白色卡片，三引擎交叉验证·多模型权重融合 |

**步骤动画时序（总时长~1.5s）：**

| 步骤 | 开始时间 | 持续时间 | 完成后表现 |

| AI分析 | 800ms | ~700ms | 状态文字闪烁「分析中」→「分析中.」→「分析中..」→「分析中...」→ ✓ 完成 |

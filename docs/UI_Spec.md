## 一、设计总则

### 1.1 核心设计原则

| 原则 | 说明 |
|------|------|
| **竖屏优先** | 375px 宽，模拟手机比例，适配抖音直播投屏 |
| **固定高度** | 手机容器强制 `height: 800px`，**永不随内容增长**。超出内容通过 `.page-container` 内部滚动展示，顶部导航栏和底部操作栏始终固定可见 |
| **暗色高级感** | Bento Grid 风格——全局深色底（Slate 900），卡片微浮于背景之上，营造"精密仪器"秩序感 |
| **大字体/高对比度** | 最小字号 14px，关键数据 ≥ 18px 加粗，暗色背景下 WCAG AA 对比度 |
| **CTA 统一** | 所有主流程操作按钮统一使用 Indigo 紫 `#818CF8`（v4.0 从红色改为紫色） |
| **手机模拟框** | 所有主播端页面包裹在 `.phone-frame` 内（顶部状态栏 + 底部 Home Bar），暗色背景 |
| **颜色即信息** | 红=冲刺，蓝=稳妥，绿=保底，紫=CTA/选中，青=数据高亮，琥珀=985/特别关注 |
| **即时反馈** | 加载<200ms 直接渲染，200ms-1s 显示骨架屏（紫色 shimmer），>1s 显示 spinner |

## 二、全局组件

### 2.1 手机模拟框 `.phone-frame`

**布局约束（直播适配关键）**：

```
┌──────────────────────────┐
│ 📶 9:41          🔋 ⚡  │ ← 状态栏 (28px, flex-shrink:0) 暗色底 #0F172A
├──────────────────────────┤
│ [导航栏按钮…]   [🔴直播] │ ← 导航栏 (flex-shrink:0, sticky) #1E293B 半透明底
├──────────────────────────┤
│                          │
│    .page-container       │ ← flex:1 + overflow-y:auto
│    (内容区，可内部滚动)    │    高度 = 800 - 28 - nav - 24
│                          │
│                          │
├──────────────────────────┤
│          ────            │ ← Home Bar (24px, flex-shrink:0)
└──────────────────────────┘
     ↑ 375 × 800px 强制固定，永不增长
```

**CSS 关键约束**：

| 属性 | 值 | 说明 |
|------|-----|------|
| `width` | `375px` | 手机比例宽度 |
| `height` | `800px` | **强制固定**，不可用 min-height |
| `overflow` | `hidden` | 截断超出内容 |
| `background` | `#0F172A` | 暗色全局底色 |
| `border` | `1px solid rgba(148,163,184,0.12)` | 微弱边框 |
| `border-radius` | `24px` | 手机框圆角 |
| `display` | `flex; flex-direction: column` | 垂直弹性布局 |
| `.status-bar` | `flex-shrink: 0` | 固定不压缩 |
| `.nav-bar` | `flex-shrink: 0` | 固定不压缩 |
| `.page-container` | `flex: 1; overflow-y: auto` | 填充剩余空间 + 内部滚动 |
| `.home-bar` | `flex-shrink: 0` | 固定不压缩 |

## 五、推荐引擎 UI 规则

### 5.0 推荐池构建（四层填充）

推荐结果由两个区域组成（对齐 PRD §4.1）：

```
📌 特别关注区（意向学校，最多3所）
   ├── 精确名称匹配（===），禁止模糊替代
   ├── 无论概率多少一律展示（0%标注说明）
   └── 不计入15所推荐总数

📊 推荐池（15所，四层填充）
   ├── L1 意向城市学校：城市→省份映射（60+城市），●强制保留，无概率门槛
   ├── L2 生源地本省学校：去重L1，按 rankProb 降序
   ├── L3 意向城市周边：邻省/邻市学校，去重L1/L2
   └── L4 全国兜底：邻省→沿海→全国，补足至15所
```

**优先级**：意向城市 > 本省 > 意向城市周边 > 全国

### 5.1 性格→院校类型倾向（tiebreaker 决胜规则）

当 rankProb 差距 ≤ 5% 时，按性格匹配度决胜（对齐 PRD §4.5）：

| 性格标签 | 优先推荐院校类型 | 逻辑说明 |
|----------|:--------------:|----------|
| 外向活泼、社交沟通、领导管理 | **综合类/文科类** | 擅长人际交往，文科/综合类院校提供更多社交场景 |
| 沉稳内敛、逻辑分析、钻研学术 | **理工科类** | 专注力强、偏好独立钻研 |
| 艺术创作 | **艺术类/设计类** | 匹配艺术类院校 |
| 动手实践 | **工科/应用型** | 偏好实操型学习 |

### 5.2 家庭经济→院校倾向

| 家庭经济 | 专业意向条件 | 优先推荐 | 逻辑说明 |
|----------|-------------|:------:|----------|
| 较为困难 | 专业方向含师范/教育类 | **师范类院校** | 师范类有生活费补贴、免学费/低学费政策 |
| 较为困难 | 无特定师范意向 | **学费≤5000元/年公办院校** | 标注"经济友好" |
| 一般/良好/比较优越 | — | 无特殊限制 | 按正常权重评分 |

### 5.3 防主播倒卖机制（UI 提示）

PDF 报告含 **5 种水印防伪**（对齐 PRD §页面⑥-五）：

| # | 水印类型 | 位置 | 内容 |
|---|----------|------|------|
| 1 | 报告编号水印 | 封面页眉/正文页眉 | `GK` + 年月日时分秒 + 随机码 |
| 2 | 防伪声明 | 封面底部/正文页眉 | "严禁倒卖 · 本内容由高考志愿规划师提供" |
| 3 | 斜纹背景水印 | 全页背景 | "高考志愿规划师 · 严禁倒卖 · [时间戳]"，45°斜角，8%-12%透明度 |
| 4 | 考生信息卡片 | 封面 | 报告编号+生成时间+网名+省份/分数+位次+选科/调剂 |
| 5 | 核验二维码 | 封面右下角 | 含校验信息（报告编号+考生信息哈希），扫码验证真伪 |

**相似度检测提示**：当新报告与主播历史报告（相同省份 + 同分数段±5分 + 相同意向学校）高度相似时，前端弹出提示：
> "⚠️ 系统检测到此报告与你 X 分钟前为另一考生生成的报告高度相似，请确认考生信息无误"

### 7.6 方案切换机制

所有方案共享同一套 HTML 结构和交互逻辑。切换只需替换 CSS 变量块：

```
方案A→B: 替换 :root 变量块 + 导航栏样式
方案A→C: 替换 :root 变量块 + 导航栏样式 + 按钮样式
方案A→D: 替换 :root 变量块 + 导航栏样式
```

生产部署时，方案B/C/D 的原型文件 `prototype_v*.html` 中 `<style>` 块即为可直接替换的 CSS。

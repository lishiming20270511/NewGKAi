## 三、意向学校功能

### 3.1 输入交互
- 用户最多填写 **5所** 意向学校
- 前端采用**模糊搜索下拉**（输入2个字符后实时搜索DB中的院校名称）
- 找不到时前端提示：**"未找到该学校，请确认名称"**，不允许提交未匹配的学校名

### 3.2 分析逻辑
针对每所意向学校，在用户**所在省份**维度下分析：
1. 近3年该专业（或相近专业）的**最低录取分数线**
2. 近3年**最低录取位次**
3. 近3年**实际录取人数**（招生规模趋势）
4. **录取概率**（见第四节算法）
5. **分数/位次波动趋势**（逐年对比，含图表）
6. **专业就业前景**：全国该专业就业率+平均薪资
7. **该校该专业就业前景**：若有数据则展示，无则注明"暂无该校专项数据"

### 3.3 数据缺失处理
- **仅1年数据：** 正常出报告，注明"该专业近年新开设，仅有1年数据可参考"
- **完全无数据：** 前端在选择时即提示，不进入报告流程
- **相近专业替代：** 若精确专业无数据，系统匹配同一学科门类下相近专业数据，并注明"以[相近专业]数据作参考"

## 四、录取概率算法

### 4.1 计算公式
```
加权最低位次 = 2025年最低位次 × 0.5 + 2024年最低位次 × 0.3 + 2023年最低位次 × 0.2

位次差值 = 加权最低位次 - 学生省内位次
（正值 = 学生位次优于历史最低，负值 = 学生位次差于历史最低）

概率映射：
  位次差 > 5000              → 95%（冲刺有把握）
  位次差 2000-5000          → 80%
  位次差 0-2000             → 65%
  位次差 -2000-0            → 45%（稳中有风险）
  位次差 -5000 至 -2000     → 25%
  位次差 < -5000            → 8%（建议作为冲刺，风险极高）
```
> 注：极低概率也显示具体数值（如8%），不做"建议不填"的文字替代

### 4.2 展示方式
- Pro/VIP 均展示精确百分比 + 可视化图表（柱状/雷达图）
- VIP 额外展示：3年位次趋势折线图 + 概率置信区间说明

1、假设学生是河南的考生，学生成绩是350分，意向城市是上海。同时他也填写了意向学校是河南科技大学、洛阳师范学院和北京大学。那么推荐的15所学校里面一定要有这3所学校。但是由于学生的成绩不可能被北京大学录取的。所以推荐的学校说明的时候，直接标注录取率是0%。另外2所学校要根据近1年或者多年（以数据库现有数据为准）
来评估录取率，具体算法需要AI去设计。在这3所学校的基础上，再从上海选12所学校。假设上海能满足该生要求的只有4所学校，那么剩余的8所学校要优先从考生生源地河南选，其次从上海周边城市选。
2、假设学生是山东的考生，学生成绩是550分，意向城市是南京，如果他没有填写意向学校的话，优先从南京地区和他成绩匹配的高校选择。一方面分数上要有一定的匹配度，这方面权重最大，另外也要综合考虑学生兴趣爱好、专业方向、家庭经济情况等综合评估考核推荐。比如假设这个学生家庭经济情况特别困难，希望将来当老师。那么可以优先考虑师范类
院校，因为师范类会有一些生活补贴之类的。假设这个学生性格很活泼开朗，在同样概率或者概率相近能考上大学的两个高校选择，优先推荐偏文科的。反之，如果这个学生性格很内向，那么优先给他推荐偏理工科的。
